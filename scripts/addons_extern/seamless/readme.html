Introduction

Piles'N'Tiles provides various tools to aid the creation of seamless meshes and textures via object management in a user defined seamless area. Objects added to this seamless area will then appear to wrap around the edges they encroach by utilizing script managed copies of the object known as seamless proxies. These proxies utilize constraints to copy the transform of the seamless object while maintaining an offset to complete the illusion of wrapping around. The results can be rendered out with an orthographic camera to create various texture maps for use in game and render engines, or other DCC software.

Although seamlessness can easily be achieved manually with dupligroups, dupliverts, and other forms of instancing, this manages individual objects in order to support seamless rigid body simulations. This allows a sizable collection of rigid bodies to be seamless before they even hit the ground. The background of this page was created with the aid of these tools. Various combinations of junk were turned into rigid bodies, made seamless, and finally dropped. This process was repeated multiple times to keep object counts from becoming too high. The texture was rendered at the final resting position.

Most of the examples displayed in this documentation were rendered straight with Cycles, full lighting included, but most will prefer to render out the usual texture maps: base color, normal, ambient occlusion, roughness, etc. Below is an example of the various rendered maps from the background texture tiled 2x on a plane in Cycles with two light sources:

Aside from simply managing seamless objects, a few extra conveniences are provided: a toggle-able rigid body margin and floor for the seamless area, automatic creation of an orthographic camera with render resolution adjustment, a seamless object compiler capable of organizing objects into dupligroups and/or merging them into a single mesh object, and a set of node groups for Cycles.

Limitations

Currently the seamless tools manage one object at a time. It is not aware of object relationships or dependencies. When an object is added to the seamless area, any proxies created are simply copies of the object via the object's copy() method. Certain object alterations, such as adding modifiers, particle systems, changing properties, etc., will not be propagated to currently existing proxy objects. Most object settings should be finalized before adding it to the seamless area. Constraints are removed on copy since seamless proxies use their own. The exception is rigid body settings. Rigid body settings can be changed after an object is added and upon refresh will propagate the settings to its proxies.

There can be only one seamless area per blend file. Originally I thought of allowing multiple seamless areas defined in a single blend file, but after witnessing the performance drop that occurs with large numbers of objects, I decided against it. This is something I would like to implement in the future should this performance issue be remedied.

Please keep in mind that these tools are a work-in-progress. All object types, except cameras, can be added to the seamless area. Some will not work properly or may result in undefined behavior. Just the same, some object settings may result in undefined behavior. Not all possible combinations have been tested.

About

I had been dumping rigid bodies in piles and began wondering if there was a good way to dump rigid bodies in piles that were seamless; either to be rendered out for textures or made into a tile-able mesh. I wanted piles and tiles! Quite possibly tile-able piles of tiles. Experimentation found use in passive rigid body copies, surrounding and mirroring the active rigid body, employing constraints. This worked out reasonably well but was horrible to set up manually. Thus began the development of these tools.

Tools and Settings

Seamless Area3D View (Object Mode) > Tool Shelf > Seamless Tab > Area

This is where the dimensions and related properties of the seamless area are defined. The seamless area will not be visible in the viewport unless its root object exists and area visibility is enabled. Visibility options of the seamless area and its subordinates are described in the display settings summary. The seamless area root will be created if it does not exist when: an object is added, the rigid body floor or margin is enabled, or the seamless camera is created. When the root object is created it is located at the world origin, not the 3D cursor. The root object is nothing more than an empty named ~seamless_area for which to parent seamless objects and dependents. The seamless area can be positioned and rotated by manipulating this object (though scale is not currently supported).

Dimensions: These are specified in whole Blender units for simplicity's sake. Any change made to the dimensions should update most dependents that rely on it such as rigid body margin and floor, proxy locations, orthographic camera scale, render resolution, and the seamless transform node group.

For seamless objects that have clipping enabled, dimension changes may still require a refresh of said objects to properly recalculate clipped proxies.

The seamless axes can be toggled here as well, but any change to them requires a manual refresh of all seamless objects. This is useful for restricting tiling to just one axis.

NOTE: Disabling an axis only restricts the seamless proxies along that axis and has no bearing on other dependents (margin, floor, camera, etc.). Disabling both axes effectively removes all proxies.

Margin: Also toggle-able in the area panel is a rigid body margin which limits how far a rigid body can leave the seamless area. This automatically re-sizes with the seamless area and is parented to it. The viewport visibility toggle is in the display panel, but the margin is only drawn if the seamless area is visible as well. As a result of the parenting, the margin transforms with the seamless area. This may be undesirable for setups involving rotated seamless areas or other complexities where rolling your own rigid body containment solution may be best. Seamless margin is simply provided as a quick convenience.

The margin is a method of keeping active rigid bodies contained, especially during outbreaks popcornitis. It is recommended to keep the margin at least big enough to encompass the largest active rigid bodies in the seamless area. If the margin is too small, objects have a higher chance to come to a rest against the margin instead of the seamless proxies. This obstructs even distribution of rigid bodies encroaching the edge, creating possible gaps in the wrap-around effect.

TIP: If the absolute collision of the margin is undesirable, force fields are an excellent alternative for proper containment.

Floor: A supplement for the rigid body margin, a rigid body floor is also conveniently toggle-able for fast rigid body pile setups. The corresponding value field specifies the floor height relative to the root of seamless area. Just like the margin, it is updated with the seamless area dimensions and is parented to it. Its visibility is also set in the display panel.

Primary Tools3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

These tools and related settings are used for adding, updating, and removing objects from the seamless area. The properties here have varying scope of which tools they affect. Adding and refreshing seamless objects are among the most expensive operations to perform depending on the total number of objects in the scene. The more objects in the scene the slower these tools get. Because of the performance implications of large numbers of objects, it is recommended to manage a seamless area in a standalone blend file, separate from any primary scene the result is to be used in.

Adding: To add selected object(s) to the seamless area, use the button here or Add Seamless from the command search menu. When an object is added to the seamless area it is placed in the ~seamless_area object's hierarchy, added to script managed groups, and based on the settings summarized in this section, its proxies are created or destroyed as appropriate. If the object is already seamless, adding is the equivalent of a refresh while assigning the proxy layers and clipping state settings below. This allows easy duplication of current seamless objects.

TIP: Turn off relationship lines in the display panel of the 3D view's property shelf. Unfortunately, Blender draws all relationship lines when this is enabled rather than just those of selected objects. Seamless objects and proxies combined with parenting can sometimes make the scene difficult to view through the relationship lines as their numbers increase.

Object Count:Maintaining Performance

Keeping the object count low is key to performance, not only for the scene, but for rigid body simulations as well. The highest object count I have personally tested with these tools was around 14k objects with the gold coin experiment. With this many objects, scene updates become very slow. With rigid body simulations, object count, active vs passive, and the collision primitives specified can drastically affect simulation performance.

Here is a few techniques for keeping performance up:

Use Dupligroups: If you are not adding rigid bodies or interactive objects to the seamless area, the best method is more than likely to just place said objects in a group, create an instance of the group, then add only the group instance to the seamless area.

Every time another non-interactive object needs to be added, just append it to the group and position as needed.

Dupligroups are also handy if you are using high poly meshes as rigid bodies. For each unique high poly object, create a simplified shell mesh to be used as the rigid body and enable dupligroups on this object, specifying the group with the high poly object. Many objects using the simplified shell instead of the actual high poly mesh saves a huge amount of memory as well as performance.

2x2 Offset: Place objects in one of the corners of the seamless area with clipping enabled. This ensures only 3 seamless proxies are created. An excellent combo for dupligroup instances, particle systems, and other static objects.

Cumulative Simulations: For rigid bodies, avoid adding all rigid body objects and running the simulation in the seamless area at the same time. Instead, start with a small selection, such as the heavy or large objects. Add these rigid bodies to the seamless area and drop them. Once the simulation is complete, apply the transform of these objects at the desired frame of the animation and convert them to passive. Once converted to passive, ensure the proxy clipping option is enabled and refresh the seamless objects. This will remove all proxies that are not encroaching the seamless area and margin. At this point repeat the process with the next batch of objects.

To reuse objects in the first batch: duplicate them, convert them to actives, and refresh the selection to generate their proxies.

Depending on your goal, you may wish to start with the large objects and work your way to the smaller ones or vise-versa. The stacking of objects may not be physically accurate using this technique, but the performance savings and control may be worth it.

To the possible detriment of rigid body performance, but increased accuracy, you may also want to change the collision primitive on the passives to something more precise as they accumulate. If you do, be sure to refresh all of the passives to propagate the change to their proxies.

Cumulative Rendering: Rigid body simulations or not, there may be cases where huge numbers of objects need to be stacked or layered, but the objects are too small or too many to efficiently manage. A good example would be gravel: Incredibly high number of small rocks resting on progressively smaller rocks.

Drop a combination of objects of approximately similar scales, maybe randomized a bit. In the case of gravel, drop a collection of rocks. For the first iteration use an appropriate ground material. Render out the required texture maps (normal, ao, etc.).

Reset the rigid body simulation, re-randomize the objects, including any required seeds used by the object materials. Replace the ground material with a new one utilizing the texture maps previously rendered out. Scale these texture maps to 2, 3, etc. times smaller than the seamless area to ensure proper alignment with no seams. Re-drop the objects and re-render the texture maps. Rinse. Repeat. Each iteration uses the previously rendered results as the new ground material with smaller scale to fill in the fine detail.

An alternative to reusing previous renders as the ground/backdrop material is to simply render out the maps with transparency and manually scale and merge them in Photoshop. It may be more or less work depending on your needs/workflow.

Removing: To remove selected object(s) from the seamless area, use the button here or Remove Seamless from the command search menu. Removing an object cleans up its proxies and clears it from the ~seamless_area object's hierarchy as well as the script managed groups.

Proxy Layers: When adding seamless objects, this sets the number of layers of proxy objects to create (clipping state permitting). Setting more than one layer is handy for objects that have dimensions that extend beyond those of the seamless area as well as objects who’s affect extends as much: light sources, objects contributing to AO/GI, etc. This can be changed manually on a seamless object by setting the object's custom property named seamless_level to the desired value and then refreshing the object.

Particle Systems: When enabled, particle systems on proxies are handled according to the particle system's seamless mode. The mode is specified in the particle system properties page below the type and seed settings.

If this is disabled, all particle systems are cleared from seamless proxies as they are duplicated. In this case, ensure that these particles are hidden at render time to preserve seamlessness. This is handy for situations where you simply wish to reference the particle system elsewhere, such as a particle instance modifier utilized in the technique explained below.

Whether or not particles appear the same on duplicate objects with matching seeds depends greatly on a variety of particle system settings. Some settings work while others fail and some cannot be used if others are. A few only work if the particle system settings are duplicated and only have one user.

Seamless Modes:Handling Particle Systems

A seamless mode property is defined in each particle system settings that dictates what happens with that particle system on newly created seamless proxies. This mode must be set before the object with the particle system is made seamless. To change the mode of a particle system already in the seamless area, the users of the particle system must be removed and reinserted to the seamless area. Currently seamless object updates do not sync particle systems between an object and its proxies.

Duplicate: Particle systems are copied, their settings are duplicated for each proxy and a driver is added to the seed value to stay in sync with the seamless object. Because each particle system on each proxy uses its own particle settings data-block, the settings should be finalized before making seamless, though the refresh button does sync the particle count and distribution properties.

Use this when you want to have random element order enabled. Having physics enabled appears to not work unless the particle system type is set to hair and even then, some properties may not work.

Linked: The particle systems are copied as-is and a driver is added to the seed value to stay in sync with the seamless object. In addition, random element order is turned off. Because all the proxies are linked to the same particle settings data-block, changes can easily be made after insertion to the seamless area.

This is handy when emitting from a single element mesh, such as a flat quad filling the seamless area. Again, some properties may not work.

Remove: This removes the particle system from seamless proxies as they are created.

None: This mode takes no action. The particle systems are copied as-is. No drivers or automated changes are applied.

NOTE: Despite the seed being synced with drivers for both linked and duplicate modes, the update via drivers refreshes the particle's random size, rotation, etc. but fails to refresh the particle's location. Unfortunately this must be done manually. Changing a property in the particle system settings usually does the trick, even when reassigned the same value. A refresh button located next to the seamless mode does this.

Dupli Particles:Alternative to Using Particle Systems Directly

Although particle systems can be synchronized between seamless objects and their proxies, an unpredictable mishmash of options must be configured precisely to successfully wrap around seamlessly.

An obvious solution of course is to make duplicates real and add them to a group then add a group instance to the seamless area. The down side is that this is a destructive workflow. You cannot easily tweak the particle system's settings and have the changes automatically reflected. In addition, this can drastically increase the object count of the scene depending on your particle numbers.

The workaround is to utilize particle instance modifiers on single triangle meshes with duplifaces enabled and a group instance child for each unique particle object type. Umm... WHAT?

For each unique object emitted from each particle system:

1. Unique Group: Add the object to its own unique group and place it at world origin. If a single particle represents multiple objects, the same applies: add all objects used by a single particle to their own unique group and place their relative center at world origin. The world origin relative to the object(s) being the emission point.

2. Triangle Mesh: Create a single triangle mesh with the triangle's area scaled to exactly 1.0 or duplicate should you already have one. Easiest way to create one is in object mode: Add > Mesh > Circle > Set vertices to 3, fill type to ngon, and radius to 0.8774.

Ensure this mesh is at the world origin.

Add a particle instance modifier to the single triangle object (if it does not already have one) and point it to the source particle system. Make sure Size is checked (and Children, if desired) under the Create From: section of the particle instance modifier's panel.

In the Duplication panel of the triangle object's properties select Faces to enable dupliface instancing. Ensure Scale is checked and Inherit Scale is 1.0.

3. Unique Group Instance: Add an instance of the group created in step 1 and parent it (without inverse) to the triangle object from step 2. The default hotkeys for parenting without inverse is Shift + Ctrl + P.

After parenting, instances of the particle objects should be emitting from the triangles of the triangle object.

The reason we parent a group instance to the triangle object instead of directly parenting the objects themselves is because rotation of our dupliface instances goes all fubar. The other benefit of doing this is we can adjust the particle object's spawn location and rotation offset. Either by manipulating the object(s) in the unique group relative to world origin, or by manipulating the single triangle itself.

4. Seamless Particle Group: If this has not already been done: Create a single seamless particle group and add an instance of it to the seamless area (at the area's origin of course). This only needs to be done once.

Add the triangle mesh from step 2 and group instance from step 3 to the seamless particle group.

Source particle systems can be located on anything; either seamless objects or not. The catch is that each unique randomized object needs to have its own particle system with unique seed and the above steps need to be repeated for each. For example, if you have 4 different grass meshes that you want to randomly spawn, you need 4 particle systems with different seeds (so they don't appear at the same location) and repeat the above steps for each one.

If you use this solution, source particle systems also are not directly rendered so you can set the render type in the particle system's render panel to None or leave it at halo so you can still see the locations (and Cycles does not render them).

Although you may now have multiple particle systems to manage, each one can be adjusted independently for each unique emission object, allowing for a non-destructive workflow of particles existing seamlessly with randomized attributes functioning correctly.

Proxy Clipping: If this option is enabled, proxies will only be created if they are encroaching the seamless area and margin (if the margin is enabled). Encroachment is found using a simple AABB intersection test. Clipping is a great way to keep object count low and maintain reasonable performance. For active rigid bodies and a few other types of objects or object properties, clipping is ignored and all proxies are created regardless of this setting. Similar to proxy layers, this option can be changed manually on a seamless object by setting the object's custom property named seamless_clip to True or False and then refreshing the object.

Position Optimization: If a proxy's encroachment is greater than that of the seamless object it represents, the seamless object is moved to its location. This option is used by all operators that utilize a seamless object update, including some operators not listed in this panel. A good use for this is to ensure a compact mesh or group instance before compiling seamless objects.

New Proxy Visibility: These are the default visibility settings used for any newly created proxy objects resulting from any operators that utilize a seamless object update. In order they are: visibility, selectability, and renderability (as listed in the outliner), and draw type (as listed in the object display panel). This has no bearing on currently existing proxy objects. To change these visibility properties on current proxies, either for all seamless objects or only those selected, use controls in the proxy visibility panel.

Refreshing: To refresh all seamless objects click the refresh button or to refresh only selected seamless objects click the cursor button next to it. This operator is listed as Refresh Seamless in the command search menu. Refreshing recalculates seamless proxies and independently cleans up any rogue proxy objects. Rogue proxies can result either from a user deleting a seamless object or a bug in one of the seamless tools (I make no claim that these tools are bug free or perfect in any way).

Selection Tools3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

Tools for easily selecting, deselecting, and inverting selections of seamless objects as well as quick selection of the seamless area root object and placement of the 3D cursor at the root object's location. They are listed as Select Seamless, Deselect Seamless, Invert Seamless Selection, Select Seamless Area and Cursor to Seamless Area in the command search menu.

Secondary Tools3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

Additional operators for manipulating seamless objects; either all or only those that are selected. These include applying visual transform, converting active rigid body objects to passive, and toggling seamless proxy clipping. Their respective entries in the command search menu are Apply Seamless Visual Transform, Active Seamless to Passive, and Set Seamless Clipping.

Proxy Visibility3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

Tool to change the visibility, selectability, and renderability (as listed in the outliner), and draw type of already existing seamless proxy objects. This can be done to all proxies or only those of the selected seamless objects via the buttons in this panel or Set Seamless Proxy Visibility in the command search menu. Because this only operates on seamless proxies, there are no expensive updates involved for the seamless objects. These are a copy of the same options listed for new proxies in the primary tools, to provide an independent way of dealing with proxies that already exist. Even if proxies are hidden, their location is still displayed as a point if the proxies option is enabled in the display panel.

Compilation Tool Experimental3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

Utility for organizing seamless objects into a group (for dupligroup usage) and/or combining compatible objects into a single mesh object. To compile use the button in this panel or Compile Seamless Objects in the command search menu. Click the arrow next to the compile button to toggle display of compile options. Because of the experimental nature of this tool, caution is recommended (have backups, save before use, etc.), especially with the mesh compiler.

The original idea behind this tool was too create an easy way of removing objects from the seamless area, group them, and then reinsert them as group instances. This was primarily to reduce object count once work was finished with any rigid body sims. The number of options has increased since then, but nothing here is finalized. This is still a very experimental work-in-progress.

Compile Options: Here you can specify the seamless compile type to either group or mesh, the name of the resulting group or mesh, and the layer(s) in which to place the result. The seamless area can be completely cleaned up and all associated objects removed. Rigid body settings of resulting objects can be cleared. The compiled result can be reinserted into the seamless area, assuming it is not set to be cleared.

If reinsertion is enabled, groups are reinserted using an empty with a dupligroup instance and meshes are reinserted as-is. If no layer is specified the result stays on the same layer(s), and if reinserted, reinsertion uses the currently active layer(s). It should also be noted that reinsertion uses the current settings specified under primary tools for adding objects to the seamless area (proxy layers, clipping, etc.).

The compiled group can be used as-is for instancing/dupligroups, tiling each instance as many times as needed. This however, is limited to flat arrays of the instances. If distortion is needed, such as stones making up a curved pathway, compiling a mesh is recommended.

Mesh Options: Mesh compilation combines all seamless mesh objects into a single mesh object. This alone can just as easily be done by converting to mesh and manually joining the seamless objects, but this tool provides a few more options.

Use Group: The compiled mesh is placed in a group, and if reinsertion is enabled, the group instance is reinserted rather than the mesh object itself.

All Objects: The compiled mesh and all other seamless objects are placed in a group.

Convert Non-Mesh: If an object can be converted to a mesh (text, curves, etc.) it will be included in the compiled mesh.

Vertex Colors: Toggle the creation of vertex color layers with random colors assigned to each unique object that makes up the mesh. When enabled the name of the layer can be specified below the checkbox. When using the generated vertex colors and UVs, keep in mind Blender's limit on the total number of each type of data layer, especially if the objects being joined are already have them.

UVs: Toggle the creation of UV layers containing various information: random values, object location, and object space. When enabled the name of the layer can be specified below the checkbox, as well as related options for each setting. Object space and location can use either one or two uv layers depending on if the Z value is needed. If the Z value is needed, the remaining channel of the second UV layer can be set to a random value or an incrementing unique value. Random values are limited between 0.0 and 1.0.

Cleanup3D View (Object Mode) > Tool Shelf > Seamless Tab > Tools

Tool to remove all objects from the seamless area and a tool to remove the seamless area itself. These are listed as Remove All Seamless and Cleanup Seamless in the command search menu.

These properties toggle the visibility of the seamless area and related objects as represented by their special appearance in the viewport. The colors associated with these items may be customized in the add-on preferences panel.

Seamless Camera3D View (Object Mode) > Tool Shelf > Seamless Tab > Camera

The tools here are used to manage an orthographic camera associated with the seamless area. The operators to create or destroy the camera in this panel change with its existence and are listed as Create Seamless Camera or Delete Seamless Camera in the command search menu. Additionally tools to flip the camera and reset it; listed as Flip Seamless Camera and Reset Seamless Camera in the command search menu.

Rendering Textures:Random Thoughts

For certain texture maps such as normal and ambient occlusion, color management needs to be turned off. This is done in the color management panel in the scene properties.

Some render passes in Cycles are either incomplete or not transformed to the desired space. For example, where the AO pass correctly takes into account material displacement output and normal texturing, the normal pass only utilizes displacement socket of the material output, ignoring any normal inputs on the various shader nodes that use it. In addition the normal pass is not transformed into camera space.

One strategy to work around the limitations of render passes is to ensure that all materials for your scene utilize a global output node group. Doing so allows you to globally override material outputs while maintaining access to unique material information. For example the junk pile background for this page uses an output node group employing a metallic/rough physically based model. By simply editing this node group, rendering out the required texture maps is easy.

To render the normal map, the normal input of the group is run through a world to camera vector transform node, scaled and offset with a mapping node, and finally plugged into an emission shader node. To render the output, this emission node is plugged into the surface output, replacing the default.

The normal map channels can be easily inverted by setting x and/or y scales to negative and swapped by setting z rotation to +/- 90 degrees.

With the normal map example, color management is turned off of course, but the other maps are done similarly, they are plugged directly into an emission output. The emission output is used for simplicity's sake as there are currently no custom data outputs and passes available for Cycles.

Contains a link to the very documentation you are viewing right now. Link can also be accessed as View Seamless Tools Documentation from the command search menu. A link to this documentation is also located in the add-on preferences where there is an option to toggle this panel as well.

Node GroupsNode Editor (Shader) > Property Shelf > Seamless Tools

A few node groups for Cycles are included with this add-on to ease seamless material creation and provide an abundance of alternative uses. The node groups can be loaded and refreshed from the property shelf here and globally by finding Load Seamless Node Groups and Refresh Seamless Node Groups in the command search menu. The approximate object location resolution entry shows up in this panel when the Seamless Area node group exists for ease of access to this internal property. Its use is detailed in the Seamless Area node group summary below.

In the context of these tools the Seamless Area node group is the primary group of interest. It internally utilizes all the other node groups except Fast Noise which is included to compliment its use.

NOTE: Although these node groups are intended to be used in Cycles, Round Vector, Round Vector Ex, and Fast Noise work correctly in BI. The Seamless Transform node group can be made to work in BI by replacing the Separate XYZ and Combine XYZ nodes with their RGB counterparts. At the time I did not realize BI did not get the XYZ versions. Unfortunately object location is not provided as an input in BI, making a compatible Seamless Area node group impractical.

Seamless Area: Provides coordinates in seamless space of object location, approximation of object location, and texture coordinates. The seamless area node group is intended to help create shaders for seamless objects and their proxies so that their material and/or textures appear the same on all of them despite being separate objects.

For all seamless area outputs X and Y tiles with the seamless area and Z coordinate extends infinitely from the origin (positive and negative). X and Y start at 0.0 in the lower left and end at 1.0 in the upper right of the seamless area.

Object Location:Vector

The object location output provides matching object coordinates between a seamless object and its proxies. This is useful for creating values that apply to the entire object. Ideally this would be accomplished with custom object properties that would be copied when the proxies are created, but unfortunately Cycles provides no way to input a custom object scoped property. The only object wide properties Cycles provides are unique to each object or based on externals like render layers. Doing this with UV or vertex color layers is a waste of memory as the same value is stored for every face loop of the mesh and different objects would need unique data-blocks.

Warning: Using object location to define material appearance is not recommended for animations or animated objects with motion blur enabled. Objects should generally be static for rendering unless you're specifically after the effect this produces. If animation or motion blur is needed, a UV or vertex color solution may be required.

Object location output visualized through the seamless camera:

A good use of object location is adding or subtracting offsets to textures such as those using object space coordinates. This can be used to prevent obvious patterns from showing up in the same location on matching objects. The intensity of the offset may need to be scaled depending on its use.

Object location may also be used directly as texture coordinates. This could be used to match colors to a texture mapped to the seamless area or possibly pseudo random values by scaling with large numbers or values of another texture.

NOTE: These coordinates should be used on textures with smooth interpolation between neighboring values or operations where precision errors do not produce noticeable differences in output. This is critical to maintaining the appearance of seamlessness between a seamless object and its proxies when encroaching the edge of the seamless area.

Approx Object Location:Vector

This output is similar to object location, but the coordinates are approximated by rounding them within a fixed range of values. The resolution of this stepped range can be easily adjusted in the Seamless Area panel of the node editor's property shelf. The higher the resolution, the more accurate the coordinates become. This should be used for precision sensitive situations such as unfiltered textures or operations involving large values.

If usage of the regular object location results in a seamless object or any of its proxies not matching each other, use this instead. Because of the rounding involved, transformed locations of a seamless object and its proxies are more likely to be the same.

Fast Noise: Pseudo random noise function that takes a 3D coordinate and returns a value between 0.0 and 1.0. This is useful for turning constants, such as object location, into random values.

Warning: Output of this node group will most likely differ between CPU, GPU, and other hardware/software floating point implementations. NOT recommended if the intention is to combine rendered results from different machines or hardware.

Round Vector: Rounds components of a vector to nearest integer.

Round Vector Ex: Scales a vector, then applies an offset, rounds components of the vector to nearest integer, and then reverses the scale on the result. Though this is used by the seamless area node group for approximate object location, its usefulness is quite extensive. The offset is actually set at 0.5 by default which technically makes this a floor vector node instead. Change the offset back to 0.0 for round vector or -0.5 for ceil vector.
